Bubblewrap TWA — correct order (fixes: ENOENT twa-manifest.json)
================================================================

1. Use a DEDICATED folder (keeps android/ + twa-manifest.json out of the git repo):

   mkdir -p ~/shoptify-twa && cd ~/shoptify-twa

2. Run INIT first (creates twa-manifest.json + android project):

   npx @bubblewrap/cli@latest init --manifest https://cmgroups.vercel.app/manifest.webmanifest

   Answer prompts: package id com.cmgroups.shopnow, app name Shoptify, signing key when asked.

3. Then BUILD:

   npx @bubblewrap/cli@latest build

   (Or: bubblewrap build — if @bubblewrap/cli is installed globally.)

Outputs: app-release-signed.apk and app-release-bundle.aab in this folder.

---

JDK "self-signed certificate" when Bubblewrap downloads JDK:
- Install JDK 17 yourself, then run init again and choose No to auto-JDK.
- macOS (Homebrew): brew install openjdk@17
  Then PATH to JDK home is usually:
  /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home
  (Intel Mac may use /usr/local/opt/openjdk@17/...)

Docker (all deps pre-installed, no local JDK/SDK):

  docker run --rm -it -v "$PWD:/project" -w /project ghcr.io/googlechromelabs/bubblewrap:latest init --manifest https://cmgroups.vercel.app/manifest.webmanifest

  Then:

  docker run --rm -it -v "$PWD:/project" -w /project ghcr.io/googlechromelabs/bubblewrap:latest build

(Run those from ~/shoptify-twa after mkdir and cd.)

Gradle: "Several environment variables ... different paths to the SDK"
========================================================================
You have BOTH ANDROID_HOME (Bubblewrap) and ANDROID_SDK_ROOT (Android Studio)
set to different folders. Gradle allows only ONE.

Before `bubblewrap build`, in the SAME terminal, run ONE of these:

Option A — use Bubblewrap’s SDK (what Bubblewrap installed):

  unset ANDROID_SDK_ROOT
  export ANDROID_HOME="$HOME/.bubblewrap/android_sdk"
  npx @bubblewrap/cli@latest build

Option B — use Android Studio’s SDK (often more complete):

  unset ANDROID_SDK_ROOT
  export ANDROID_HOME="$HOME/Library/Android/sdk"
  npx @bubblewrap/cli@latest build

Do not set both ANDROID_HOME and ANDROID_SDK_ROOT to different paths.

To make this permanent, remove or comment out ANDROID_SDK_ROOT from
~/.zshrc if you added it, and keep only ANDROID_HOME.

TWA shows URL bar (looks like a website) — Digital Asset Links mismatch
========================================================================
The package name in ~/.shoptify-twa/twa-manifest.json ("packageId") MUST match
one of the "package_name" values in:

  frontend/public/.well-known/assetlinks.json

If Bubblewrap generated app.vercel.cmgroups.twa but assetlinks only listed
com.cmgroups.shopnow, verification fails and Chrome shows the browser UI.

The repo assetlinks.json includes BOTH common IDs with the same SHA-256
fingerprint (same keystore). After changing packageId, rebuild the APK or
add another statement to assetlinks.json and redeploy Vercel.
